# 如果电话亭

文旅策划创新组合工具。不是助手，是道具——拿起来就用，用完就放。

核心洞察：**文化都是API，公平的；嫁接方式不同，才有价值。**

---

## 核心公式

```
项目地 A（一个具体文化元素） × 嫁接策略（8选） = "如果A做B..."
```

输入两样东西：一个项目地 + 一个项目类型。
输出：3-5张方案卡片，每张 = 一种嫁接策略 × 你的文化元素。

---

## 五步法工作流

### Step 1：深挖文化素材（8维度 × 深度挖掘）

**核心问题：浅挖 = 同质化方案。** 如果只搜到"燃灯塔、2248枚铜铃、鲶鱼传说"这些百度百科级别的素材，出来的方案一定平庸。必须挖到**当地人知道但游客不知道的层面**。

#### 1.1 搜索策略（强制执行，每个维度单独搜索）

**禁止一次性搜索整个项目地。** 必须按维度分别搜索，每个维度用不同的搜索关键词：

| 维度 | 搜索策略 | 禁止出现的词 |
|---|---|---|
| 地理自然 | 搜：`[项目地] + 地质 / 水文 / 气候 / 地貌 / 植被 / 动物` | 禁止搜"风景""旅游" |
| 城市功能 | 搜：`[项目地] + 历史功能 / 旧称 / 曾用名 / 行政沿革 / 产业变迁` | 禁止搜"文化""旅游" |
| 历史人文 | 搜：`[项目地] + 考古 / 出土 / 碑刻 / 地方志 / 族谱 / 野史` | 禁止搜"历史故事" |
| 神话传说 | 搜：`[项目地] + 方志 / 民间故事 / 神灵 / 禁忌 / 仪式 / 方言俗语` | 禁止搜"传说""神话"（太泛） |
| 当代文化 | 搜：`[项目地] + 夜市 / 广场舞 / 直播 / 摆摊 / 广场 / 社区活动 / 年轻人` | 禁止搜"文创""网红" |
| 产业工艺 | 搜：`[项目地] + 作坊 / 工匠 / 原料 / 工序 / 工具 / 废料 / 配方` | 禁止搜"非遗""手工艺" |
| 社会记忆 | 搜：`[项目地] + 方言 / 歌谣 / 口述史 / 老照片 / 地名由来 / 街巷故事` | 禁止搜"乡愁""记忆" |
| 感官印象 | 搜：`[项目地] + 味道 / 声音 / 气候体感 / 季节变化 / 早晚差异 / 食物气味` | 禁止搜"美景""好看" |

#### 1.2 深度标准（每个维度至少 8-10 个关键词）

**每个维度必须产出 8-10 个关键词**，而不是3-5个。不够就继续搜，用以下追问法加深：
- "这个东西的**原料/来源**是什么？"（从产品挖到原材料）
- "这个工艺的**废料/副产品**是什么？"（从成品挖到废料）
- "这个传说的**反面版本**是什么？"（从正史挖到野史）
- "这个地方**最丑/最奇怪/最不起眼的东西**是什么？"（从景点挖到角落）
- "本地人**最不希望游客知道的事**是什么？"（从官方叙事挖到民间真实）
- "这个地方**已经消失的东西**是什么？"（从现状挖到历史废墟）

#### 1.3 素材分层（关键：区分"景点层"和"矿藏层"）

挖到的素材必须分两层：
- **景点层**（禁止作为方案主体）：百度百科能查到的、已被广泛使用的元素。如"燃灯塔""2248枚铜铃"——这些是**背景知识**，不是**创意原料**
- **矿藏层**（必须作为方案主体）：当地人知道但游客不知道的、需要深度搜索才能找到的、有"反常识"特质的素材。如"塔上第104尊神像是猪八戒，比《西游记》还早几十年""铜铃铭文里的错别字""塔顶自生榆树200年不死"

**规则：每个方案的"如果句"必须引用矿藏层素材，禁止引用景点层素材。**

#### 1.4 反常识素材优先

搜索时优先找以下类型的素材（按惊奇度排序）：
1. **反直觉的事实**：塔上有个猪八戒像比《西游记》还早几十年
2. **消失的事物**：曾经存在但已消失的东西（废墟、旧习俗、老地名）
3. **微观细节**：铜铃铭文里的错别字、砖雕上某个工匠的签名
4. **跨维度矛盾**：一个佛教塔上出现道教元素、一个北方城市有南方地名
5. **身体记忆**：与这个地点相关的具体身体感受（不只是"好看"）
6. **声音/气味/温度**：最容易被忽略但最能产生奇观感的感官细节

#### 1.5 跨维度碰撞挖掘（Step 1.5，强制执行）

**每个维度至少找2个与其他维度的"意外连接"。** 方法：

对每个维度的关键词，逐一问：**"这个东西和哪个维度的东西放在一起会很奇怪？"**

示例（通州三庙一塔）：
- 地理自然×神话传说："运河鲶鱼"和"塔下入海洞口"→ 鲶鱼精从洞口出入（常规）
- 地理自然×产业工艺："运河淤泥"和"铜铃铸造"→ 铜铃铭文的铜来自运河淤泥中的回收铜（反常识 ✓）
- 当代文化×历史人文："直播带货"和"漕运验粮"→ 300年前漕官验粮的流程和今天直播间验货一模一样（反常识 ✓）
- 感官印象×社会记忆："运河水的气味"和"船工号子"→ 船工闻到某种水草气味就知道快到通州了（身体记忆 ✓）

**这些跨维度碰撞点才是方案的真正原料。** 单维度素材=平庸方案，跨维度碰撞=惊奇方案。

### Step 2：检索前沿案例

检索沉浸式/文旅相关案例，目的：避免重复、找到差距、激发变异。

**降级检索策略（检索失败不阻塞流程）：**
1. 通用检索：`[项目类型] + immersive / experience / case study`
2. 特定来源：`site:blooloop.com [项目类型]`
3. 中文检索：`[项目地] + 沉浸 / 夜游 / 案例`

每个案例提取：名字 + 一句话亮点 + 与本案的差异点。

### Step 3：组合生成（8种嫁接策略）

从8种嫁接策略中**选用最适合当前A+B组合的3-5种**：

| 嫁接策略 | 定义 | 适用场景 |
|---|---|---|
| **时空折叠** | 不同时代/时空在同一体验中并置 | 有历史纵深感的地方 |
| **空间活化** | 把非演出空间变成演出/体验空间 | 有特殊建筑/自然地貌的地方 |
| **感官转译** | 把一种感官特征翻译成其他感官 | 有强烈视觉/听觉/嗅觉特征的地方 |
| **仪式化** | 把日常行为升级为仪式体验 | 有强精神/信仰属性的地方 |
| **叙事化** | 把文化素材变成可参与的故事线 | 有神话/传说/人物的地方 |
| **匠人化** | 把传统工艺变成可参与的创作体验 | 有非遗/手工艺的地方 |
| **社区共建** | 把本地居民变成体验的一部分 | 有活态社区的地方 |
| **废墟美学** | 把废弃/工业遗迹变成体验场域 | 有工业遗产/废墟的地方 |

详细定义见 [references/combination-strategies.md](references/combination-strategies.md)。

**使用规则：**
- 不强制全用，跳过牵强的策略
- 宁少勿滥——3个差异明显的方案 > 5个同质化的方案
- 每个方案必须选不同的嫁接策略（不允许两个方案用同一种策略）

#### 素材独占规则（严格执行）

**每个文化元素只能出现在一个方案中。** 生成方案前，先列出所有矿藏层素材，然后分配：

```
方案1 → 素材A（如：铜铃铭文错别字）+ 素材B（如：运河淤泥回收铜）
方案2 → 素材C（如：猪八戒像比西游记早）
方案3 → 素材D（如：通州城是船形）+ 素材E（如：消失的东关码头）
方案4 → 素材F（如：船工闻水草气味辨方向）
方案5 → 素材G（如：除夕佛灯飞向盘山）
```

**禁止出现的模式：**
- ❌ 5个方案都提到"2248枚铜铃"→ 一个元素被反复使用
- ❌ 3个方案都围绕"燃灯塔"→ 所有方案都在说同一件事
- ❌ 2个方案用了同一个传说的不同版本→ 本质还是同一个故事

**如果某个元素被多个方案征用，说明：要么素材池太小（回去补充Step 1），要么方案同质化（换策略重写）。**

#### 多样性强制检查

生成方案后，检查以下维度。**任意2个方案之间，至少3个维度不同：**

| 维度 | 必须拉开的方向 |
|---|---|
| **文化素材** | 每个方案必须使用不同的矿藏层素材（素材独占规则） |
| 嫁接策略 | 8种策略中选不同的 |
| 情绪光谱 | 震撼 / 宁静 / 幽默 / 恐惧 / 仪式感 / 童趣 / 怀旧 / 好奇 |
| 参与深度 | 纯观看 / 可触摸 / 可行走穿越 / 需对话共创 / 需集体协作 |
| 时间节奏 | 瞬间爆发 / 缓慢生长 / 需要等待 / 有仪式流程 |
| 物理空间 | 水面 / 陆地 / 建筑立面 / 空中 / 地下 / 人体穿戴 |

### Step 4：输出标准化卡片

每个方案输出一张卡片。**质量门控见下方。**

```
### 方案 X/N：【方案名】 · 【嫁接策略名】 【emoji】

> **如果【A的一个具体文化元素】做【B的一种体验形式】，会怎样？**

**组合逻辑**：
1-2句话解释A和B怎么"接"上的。必须包含一个具体的"碰撞点"——两个看似无关的事物为什么能接在一起。

**关键场景**：
• 场景1：（具体到尺寸、材质、光效、人在这个空间里的动作）
• 场景2：（同上）
• 场景3：（同上）
• 高潮moment：（体验中最震撼的一刻）

**设计亮点**：一句话说清为什么这个方案跟别人不一样。

**可视化设计要素**：
空间类型→具体空间形态
材质语言→具体材料名称（不要泛泛的"金属"，要"耐候钢""镜面不锈钢316L"）
光影策略→具体技术手段（不要泛泛的"灯光"，要"DALI调光系统0.1%精度""单束窄角射灯"）
色彩体系→具体色名（不要泛泛的"暖色"，要"暗红→橙红→亮橙→白炽→余烬灰"）
形态关键词→2-3个形态描述
效果图Prompt→一段可直接用于AI绘图的英文描述词组
```

### Step 5：询问保存路径

- 询问用户是否需要保存
- 用户说不用 → 不保存
- 用户说用 → 询问路径，保存为 `YYYYMMDD-项目地-项目类型.md`

## 质量门控（严格执行）

### 1. 如果句质量门控

如果句 = 方案的第一句话，必须制造**认知碰撞**——两个看似无关的事物被强行接在一起，让人"停一下"。

**检查方法：** 读完如果句后问自己——"这个组合我以前见过吗？"
- 见过 → 不及格，重写
- 没见过但觉得合理 → 及格
- 没见过且觉得"等等，这能接？" → 优秀

**素材检查：** 如果句中引用的文化元素，是否出现在其他方案中？
- 出现在其他方案中 → 不及格（违反素材独占规则），换素材
- 5个方案都围绕同一个地标（如"燃灯塔"） → 不及格，说明素材池没挖开

**反面教材：**
- ❌ "如果燃灯塔的铜铃声能从脚底传上来" → 感官转译，太常规
- ❌ "如果运河水面上漂浮着莲花灯" → 祈福+放河灯，人人都想到
- ✅ "如果御窑碎瓷片考古现场做森林夜游的脚下体验" → 碎瓷+踩踏+夜游，碰撞感强
- ✅ "如果壁画修复师的'最小干预'原则做数字展的互动机制" → 修复哲学+互动设计，概念碰撞

### 2. 关键场景质量门控

每个场景描述必须包含：**具体尺寸 + 具体材质 + 具体光效 + 人在这个空间里的动作**。

**检查方法：** 场景描述能否让设计师直接画出草图？
- 不能 → 不及格，补充细节
- 能 → 及格
- 能，且画面感极强 → 优秀

**反面教材：**
- ❌ "水面有LED灯光随声音变化" → 没有尺寸、没有材质、没有具体光效
- ❌ "游客走进一个黑暗空间" → 没有空间描述、没有感官细节
- ✅ "入口隧道：高亮度白光长廊，模拟敦煌正午阳光，刺激瞳孔收缩" → 有光效、有目的、有人的生理反应
- ✅ "碎瓷步道：透明树脂封装的瓷片嵌入路面，踩一步触发对应年代的釉色光效" → 有材质、有交互、有时间维度

### 3. 组合逻辑质量门控

组合逻辑 = 解释A和B为什么能接在一起。必须包含一个**"为什么是这个文化元素"**的论据——不是泛泛的"因为有文化"，而是这个元素的独特性。

**反面教材：**
- ❌ "燃灯塔有2248枚铜铃，所以可以做声音装置" → 任何有铃铛的地方都能做
- ✅ "御窑规矩：瑕疵瓷器当场砸碎掩埋，数百年积累4000万片碎瓷——这些碎片本身就是景德镇最独特的文化资产" → 只有景德镇有这个条件

## 下一步选项

生成方案后，引导用户选择：

1. **满意，保存方案**
2. **调整某个方案** → 指定方案和调整内容
3. **换一种策略再加 2 个新方案**
4. **保留项目地 A，换一个项目类型 B 重新组合**
5. **不满意，重新开始**

---

## 上下文管理

- 同一会话中不超过 2-3 轮生成
- 每轮结束后询问："需要继续新组合吗？还是先停在这里？"
- 上下文接近上限时提示保存当前方案

## 协作原则

Question-First：确认 A 和 B → 执行五步法 → 展示方案 → 下一步选项。用户始终掌控决策权。

---

## 禁止事项

| 不输出的内容 | 原因 |
|---|---|
| 对标案例的详细描述 | 案例是灵感参考，不是方案主体；详细描述会限制想象力 |
| 效果图的完整描述 | 那是设计师的工作，策划师提供"概念"，设计师提供"视觉" |
| AI绘图提示词的中文翻译 | 英文Prompt直接给，不需要翻译 |
| 大段历史介绍 | 只输出"组合所需的最小上下文"——3-5个关键词足够 |
