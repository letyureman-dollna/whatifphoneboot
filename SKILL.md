# 如果电话亭

文旅策划创新组合工具。不是助手，是道具——拿起来就用，用完就放。

## 核心公式

```
项目地 A（地方特征） + 项目类型 B（体验形式） = "如果A做B..."
```

输入只有两样东西，输出一张可直接用的方案卡片。

---

## 五步法工作流

### Step 1：深度挖掘文化素材（8维度）

从8个维度挖掘项目地 A 的文化素材，**每个维度只提炼 3-5 个关键词**：

| 维度 | 探索的问题 |
|---|---|
| 地理自然 | 自然基底是什么？ |
| 城市功能 | 这个空间原本在做什么？ |
| 历史人文 | 发生过什么故事？ |
| 神话传说 | 有什么故事可以借用？ |
| 当代文化 | 年轻人在这里做什么？ |
| 产业工艺 | 这里产出什么？ |
| 社会记忆 | 人们对这个地方有什么情感？ |
| 感官印象 | 闻起来/看起来/听起来像什么？ |

打破"只有传统文化才算文化"的偏见——朝阳公园没有古文化，但有城市功能、当代文化、社会记忆。

### Step 2：检索前沿案例

检索沉浸式/文旅相关案例，目的：避免重复、找到差距、激发变异。

**降级检索策略（检索失败不阻塞流程）：**
1. 通用检索：`[项目类型] + immersive / experience / case study`
2. 特定来源：`site:blooloop.com [项目类型]`
3. 中文检索：`[项目地] + 沉浸 / 夜游 / 案例`

任一优先级无有效结果时，跳过进入 Step 3，提示用户"案例检索暂未获取到有效信息"。

### Step 3：组合生成（5种策略 + 多样性强制）

用5种组合策略，从策略库中**选用最适合当前 A+B 组合的 3-5 种**：

| 策略 | 核心思维 |
|---|---|
| 感官转译 | 一种感官 → 另一种感官 |
| 时空错位 | 熟悉的东西放到陌生位置 |
| 尺度翻转 | 改变尺度 → 改变关系 |
| 功能嫁接 | 一个东西能做什么 → 另一个场景 |
| 叙事植入 | 故事 → 体验动线 |

详细定义和操作指令见 [references/combination-strategies.md](references/combination-strategies.md)。

**使用规则：** 不强制全用，跳过牵强的策略。宁少勿滥——3个差异明显的方案 > 5个同质化的方案。

#### 多样性强制约束（必须执行）

生成方案后，逐条检查以下5个维度。**任意2个方案之间，至少有3个维度不同**，否则判定为同质化，必须重写：

| 维度 | 必须拉开的方向 | 禁止出现的模式 |
|---|---|---|
| **感官通道** | 视觉 / 听觉 / 触觉 / 嗅觉 / 味觉 / 温度感 | 不能全是"好看"，必须覆盖至少2种感官 |
| **载体空间** | 水面 / 陆地 / 建筑立面 / 空中 / 人体穿戴 / 地下 | 不能全放在同一个物理空间 |
| **情绪光谱** | 震撼 / 宁静 / 幽默 / 恐惧 / 仪式感 / 童趣 / 怀旧 / 好奇 | 不能全是"好美/好震撼"，必须覆盖至少2种情绪 |
| **参与深度** | 纯观看 / 可触摸 / 可行走穿越 / 需对话共创 / 需集体协作 | 不能全是"看"，必须有至少1个非视觉参与方案 |
| **时间节奏** | 瞬间爆发 / 缓慢生长 / 需要等待 / 有仪式流程 / 可逆/不可逆 | 不能全是"一眼看完"，必须有时间维度的设计 |

**检查方法（生成每个新方案前执行）：**
1. 列出已生成方案的5维度取值
2. 新方案必须在至少3个维度上与每个已有方案不同
3. 如果无法满足，换策略或换文化素材维度重新组合，不要硬凑

**反面教材：**
- ❌ 5个方案全是"水面+灯光+视觉+观看+瞬间"→ 1个方案换5种说法
- ✅ 1个水面灯光（视觉/震撼）+ 1个塔内声景（听觉/仪式感）+ 1个地面盲道触觉路径（触觉/好奇）+ 1个气味花园（嗅觉/宁静）+ 1个集体漕运号子（听觉+协作/幽默）

### Step 4：输出标准化卡片

每个方案输出一张卡片，包含 6 个部分：

```
## 如果句
一句话概括核心概念（PPT标题页直接用）

## 组合策略
使用了哪种策略（方法论标签）

## 组合逻辑
为什么这样组合（说服甲方的论据）

## 关键场景
核心体验场景描述（给导演/空间设计师）

## 设计亮点
差异化卖点（提炼重点）

## 可视化设计要素
空间类型：大地装置 / 装置地景 / 漂浮平台 / 沉浸式隧道
材质语言：镜面不锈钢 / 夯土 / 再生塑料 / 透光混凝土
光影策略：投影mapping / 激光 / 荧光 / 自然光
色彩体系：CBD灰+公园绿 / 荧光色系 / 大地色系
形态关键词：流线型 / 模块化 / 像素化 / 有机形态
效果图Prompt关键词：可直接用于AI绘图的描述词组
```

### Step 5：询问保存路径

- 询问用户是否需要保存
- 用户说不用 → 不保存
- 用户说用 → 询问路径，保存为 `YYYYMMDD-项目地-项目类型.md`

---

## 下一步选项

生成方案后，引导用户选择：

1. **满意，保存方案**
2. **调整某个方案** → 指定方案和调整内容
3. **换一种策略再加 2 个新方案**
4. **保留项目地 A，换一个项目类型 B 重新组合**
5. **不满意，重新开始**

---

## 上下文管理

- 同一会话中不超过 2-3 轮生成
- 每轮结束后询问："需要继续新组合吗？还是先停在这里？"
- 上下文接近上限时提示保存当前方案

## 协作原则

Question-First：确认 A 和 B → 执行五步法 → 展示方案 → 下一步选项。用户始终掌控决策权。
